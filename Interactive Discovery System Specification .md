### Functional Questions

**Q1: CARPET Enforcement Level**
- Should system BLOCK advancement to Thesis without complete CARPET?
- Or just warn but allow override?
- Recommendation: Warn + require confirmation to override

**Q2: Question Customization**
- Should reps be able to add custom questions on-the-fly?
- Or keep questions fixed to maintain framework integrity?
- Recommendation: Fixed questions, but allow "Notes" field for custom topics

**Q3: Multi-User / Team Features**
- Is this single-user tool or will multiple reps use same instance?
- Should there be team analytics/reporting?
- Recommendation: Start single-user, add team features in Phase 2

**Q4: Session Handling**
- How long should session data persist? 7 days? 30 days?
- Should old sessions auto-archive or require manual cleanup?
- Recommendation: 30-day auto-archive with manual override

### Design Questions

**Q5: CARPET Tracker Placement**
- Persistent sidebar (always visible, takes screen space)?
- Floating widget (collapsible, less intrusive)?
- Top bar (horizontal layout, minimal space)?
- Recommendation: Test with users, likely floating widget

- Always-visible floating button?
- Keyboard shortcut (e.g., Ctrl+E)?
- Context menu (right-click)?
- Recommendation: Floating button + keyboard shortcut

**Q7: Progress Indicator Style**
- Linear bar (simple, familiar)?
- Circular/radial (modern, shows % complete)?
- Checklist (clear, traditional)?
- Recommendation: Linear bar for phases, checklist for CARPET

- Recommendation: localStorage primary, cloud optional for redundancy

- Recommendation: Modern browsers only
